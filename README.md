# m-index-lab

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库旨在集中存放各类独立编写的 HTML 页面。这些页面通常为单页应用、实验性前端代码、静态演示页面或临时性工具页面。不针对任何特定域名、网站或服务，仅作为页面资源的归档和展示容器。

## 目录说明

```
m-index-lab/
├── pages/           # 存放所有独立 HTML 页面文件
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/          # 页面共用的资源文件（CSS、JS、图片等）
├── index.html       # 仓库根目录下的索引页面（可选）
└── README.md        # 本文件
```

- `pages/` 目录是主要页面存放位置，每个页面建议独立命名，避免冲突。
- `assets/` 目录用于存放页面间共享的静态资源，若页面使用独立资源可自行创建子目录。
- 根目录下的 `index.html` 可作为页面导航或说明页，非必需。

## 页面归档说明

1. **页面独立性**  
   每个 HTML 页面应尽可能自包含，减少对外部资源的依赖。如需引用共用资源，请放置于 `assets/` 目录并注明路径。

2. **命名规范**  
   页面文件名建议使用小写英文字母、数字及连字符（`-`），例如 `my-page.html`。避免使用中文、空格或特殊字符。

3. **版本与更新**  
   本仓库不强制要求版本记录。如需更新某个页面，可直接覆盖原文件。建议在提交信息中简要说明修改内容。

4. **兼容性**  
   页面应尽量兼容主流现代浏览器（Chrome、Firefox、Safari、Edge），不保证对老旧浏览器的支持。

## 维护说明

- 本仓库由维护者自行管理，不对外提供持续集成或自动部署服务。
- 欢迎通过 Issues 提交问题或建议，但请注意本仓库不承诺响应时间或功能实现。
- 如需贡献新页面，请提交 Pull Request，并在说明中简述页面用途和来源。
- 仓库不存储任何用户隐私数据或敏感信息，所有页面内容均视为公开。

本仓库默认不设定具体开源许可证。页面内容版权归各自作者所有。如需引用或修改，请自行与作者协商。
